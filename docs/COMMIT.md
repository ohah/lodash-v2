# 커밋 규칙

커밋 메시지는 **Conventional Commits** 스타일을 따릅니다.

## 형식

```
<type>: <제목>

[본문 (선택)]
```

- **제목**: 50자 이내, 명령형 (예: "추가한다" O / "추가함" X)
- **본문**: 필요 시 변경 이유·영향 범위 작성

## type

| type | 용도 |
|------|------|
| **feat** | 새로운 기능, 문서, 스크립트 추가 |
| **fix** | 버그 수정 |
| **chore** | 빌드/설정/의존성/초기 셋업 등 코드 외 변경 |
| **docs** | 문서만 수정 (README, 주석 등) |
| **refactor** | 동작 변경 없이 코드 구조·이름 정리 |
| **test** | 테스트 추가·수정 |
| **perf** | 성능 개선 |

## 예시

```
feat: HEARTBEAT 셀프피드백, 구현 체크리스트, core 빌드(Node+Web) 및 로드 테스트
```

```
feat: es-toolkit 3-way 비교, 결과 동등성 테스트, AI 피드백 문서, 린트/포맷
```

```
chore: initial lodash-v2 bun monorepo
```

```
fix: chunk 빈 배열일 때 undefined 반환하던 문제 수정
```

여러 변경을 한 커밋에 넣을 때는 제목에 요약하고, 필요하면 본문에 항목별로 적습니다.
